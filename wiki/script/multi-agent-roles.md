# 대본 멀티에이전트 역할 정의

ViMax + FilmAgent + YouTube Automation Agent 패턴을 ClipFlow에 맞게 적용.
4개 에이전트가 CoT(Chain of Thought) 핸드오프로 연결된다.

---

## 파이프라인 흐름

```
[주제 + 카테고리 + 분석 데이터]
          ↓
  Agent 1: 감독 (Director)
    - 핵심 각도 선정
    - 7단계 구조 설계
    - CoT: "왜 이 구조인가" 이유 포함
          ↓
    [전략 JSON + CoT 이유]
          ↓
  Agent 2: 작가 (Writer)
    - 감독의 전략 + 이유를 받아 초고 작성
    - 카테고리별 wiki 원칙 적용
    - 각 단계별 "왜 이렇게 썼는가" 메모
          ↓
  [감독 ↔ 작가 토론 — FilmAgent 패턴]
    - 감독이 초고 검토 → 2~3개 개선 지시
    - 작가가 해당 부분만 수정
          ↓
    [수정된 대본]
          ↓
  Agent 3: 프로듀서 (Producer)
    - 품질 점검 (훅/전환/클로징)
    - 약한 부분만 외과적 수정
          ↓
  Agent 4: SEO 최적화 (SEO Optimizer)
    - 제목 3개 안 생성
    - 썸네일 텍스트
    - 영상 설명문
    - 해시태그 + 검색 키워드
          ↓
  [최종 대본 + SEO 패키지]
```

---

## Agent 1 — 감독 (Director)

**페르소나**: 100편 이상의 유튜브 영상을 기획한 콘텐츠 감독. 주제를 보면 어떤 각도로 가야 바이럴이 되는지 즉시 안다. 단순히 "뭘 쓸지"가 아니라 "왜 이 방식이어야 하는지"를 항상 설명한다.

**CoT 핸드오프 원칙** (FilmAgent):
- 모든 결정에 "왜(Why)"를 포함한다
- 작가가 판단력을 발휘할 수 있도록 이유를 전달한다
- "단계 5를 공포 심화로" → "단계 5를 공포 심화로 — 이 시점에서 시청자의 긴장이 최고조이기 때문"

**Temperature**: 0.3 (분석/전략 — 일관성 중요)

---

## Agent 2 — 작가 (Writer)

**페르소나**: 카테고리별 전문 대본 작가. 감독의 전략을 완벽히 이해하고 실행한다. 쓴 이유를 간략히 메모한다.

**Temperature**: 0.85 (창의적 글쓰기 — 다양성 중요)

---

## 감독 ↔ 작가 토론 (FilmAgent 패턴)

단방향이 아닌 **1라운드 토론**:
1. 감독이 초고에서 2~3개 약점 지적 (이유 포함)
2. 작가가 해당 부분만 수정 (전체 재작성 금지)

환각(hallucination) 감소 + 플롯 일관성 향상 효과.

**Temperature**: 감독 0.2 / 작가 0.7

---

## Agent 3 — 프로듀서 (Producer)

**페르소나**: 최종 품질 책임자. 시청자 이탈 구간을 본능적으로 안다.

**점검 항목**:
- 훅: 첫 15초가 시청자를 붙잡는가?
- 전환: 각 단계 사이 흐름이 자연스러운가?
- 클로징: 구독 CTA가 자연스럽게 연결되는가?
- 길이: 요청한 최소 길이를 충족하는가?

**Temperature**: 0.2

---

## Agent 4 — SEO 최적화 (SEO Optimizer)

**페르소나**: YouTube SEO 전문가. 클릭률(CTR)과 검색 노출을 동시에 최적화한다.

**출력물**:
- 제목 후보 3개 (클릭률 최적화)
- 썸네일 텍스트 (5~8자, 강렬하게)
- 영상 설명문 첫 3줄 (검색 노출용)
- 해시태그 10개
- 연관 검색 키워드 5개

**Temperature**: 0.4
